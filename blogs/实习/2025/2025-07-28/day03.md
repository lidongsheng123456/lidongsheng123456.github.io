# day03-ollama使用以及在Tauri中使用Rust作为后端调用本地大模型

## 一.概述

1. ollama是一个本地运行大模型的技术框架，简化大型语言模型的部署和管理过程。

## 二.mcp

1. mcp大模型上下文协议，它允许模型客户端与外部服务和工具服务器进行交互，让模型客户端不再局限于对话和信息检索，而是能采取实际行动，比如发送邮件，部署代码或发布文章等。

## 三.应用场景

1. 通过mcp，AI可以自己去使用我们写的方法和API，例如：读写文件、查询数据库、执行系统命令、分析图片，通过mcp让AI从“只会说话”变成“会使用工具的智能助手”。

## 四.在Tauri中有两种调用本地大模型方式

1. 直接调用：前端 》Rust 》ollama API
2. mcp协议模式：前端 》 Rust 》 mcp服务器 》 ollama API

## 五.ollama提供的Rest API

1. post		/api/generate		- 生成文本
2. get		/api/tags			- 获取模型列表
3. post		/api/chat			- 聊天对话
4. get		/api/version		- 获取版本信息

## 六.小结

1. 在rust中定义实体类并不想java一套写好，而是先使用stract定义结构体，结构体可以有多个属性，值可以是不同类型，方法要额外往下写，例：impl 结构体名称，这样就可以是一个基本对象的模版；
2. 在Rust中self的大小写有着不同作用；
3. Self（大写）：是个类型别名，代表当前实现块所属的类型，作用：作为类型返回，作为构造器，代替具体类型名称；
4. self（小写）：实例引用，用于访问实例的方法和属性，例：我用Self构造器返回一个实例，这个实例的引用用个变量接收，实例中如果有self.name，就代表在调用我们接收的这个实例变量里面的属性。
